開発環境 vs CI環境

切り替えるのは Web ポートの host_network だけ — internal は共通

開発環境
localhost でブラウザからアクセス
# /etc/nomad.d/client.hcl host_network "local" {
  interface = "lo"
}
host_network "internal" {
  interface = "docker0"
}
# myapp.nomad.hcl port "web" { static = 3000, host_network = "local" }
port "api" { static = 8080, host_network = "internal" }

CI環境
外部クライアントからアクセス
# /etc/nomad.d/client.hcl host_network "public" {
  interface = "eth0"
}
host_network "internal" {
  interface = "docker0"
}
# myapp.nomad.hcl port "web" { static = 3000, host_network = "public" }
port "api" { static = 8080, host_network = "internal" }
Host
CIクライアント
eth0 — 192.168.x.x
host_network "public"
docker0 — 172.17.0.1
host_network "internal"
docker0 bridge (172.17.0.0/16)
Web:3000172.17.0.2 0.0.0.0:3000
API:8080172.17.0.3 172.17.0.1:8080
MySQL:3306
環境間で異なる設定
client.hcl:
  開発: host_network "local" { interface = "lo" }
  CI: host_network "public" { interface = "eth0" }

Job ファイル:
  開発: port "web" { host_network = "local" }
  CI: port "web" { host_network = "public" }
環境間で共通の設定
client.hcl:
  host_network "internal" { interface = "docker0" }

Job ファイル:
  port "api" { host_network = "internal" }
  extra_hosts = ["host.docker.internal:host-gateway"]
  API_URL = "http://host.docker.internal:8080"
  DB_HOST = "host.docker.internal"
通信経路は両環境で同じ構造

アクセス元 → Web :3000 → host.docker.internal:8080 → API :8080 → host.docker.internal:3306 → MySQL

違いは Web ポートの入り口だけです。 開発環境では lo（127.0.0.1） にバインドしてブラウザから localhost:3000 でアクセスし、 CI環境では eth0（0.0.0.0） にバインドして外部クライアントから 192.168.x.x:3000 でアクセスします。 API ポートは両環境とも docker0（172.17.0.1） にのみ公開され、外部に露出しません。
