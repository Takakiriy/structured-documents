各ステップの詳細
STEP 1 — 通常の送信フロー
送信者がメールを送ると、送信側 MTA は DNS に問い合わせて宛先ドメインの MX レコードを取得し、受信側 MTA に SMTP で配送します。
STEP 2 — アドレス不在を検出
受信側 MTA が宛先アドレス（例：unknown@b.com）を確認すると、そのアドレスが存在しないことがわかります。受信側 MTA はエラーを返し、NDR（Non-Delivery Report） の生成が始まります。
STEP 3 — 1st バウンス（DSN / NDR）送信
受信側 MTA は DSN に従ったバウンスメールを生成し、元のメールの Return-Path ヘッダーに設定されたアドレス宛に返送します。このアドレスは通常、no-reply@a.com などのシステムアドレスか、場合によっては空（<>）です。
STEP 4 — ダブルバウンス発生 → 破棄
Return-Path のアドレスも無効だった場合、そのバウンスメールを届けようとするとさらにエラーが発生します。これがダブルバウンスです。RFC の仕様では、エンベロープ送信者が空（<>）のメールにはバウンス通知を送ってはならないと定められており、このメールは黙って破棄されます。誰にも通知されず、問題の追跡が困難になります。
用語解説
MX
Mail Exchanger レコード
DNS MX record
DNS に登録された情報で、「このドメイン宛のメールをどのサーバーが受け取るか」を指定します。メール送信時、送信側 MTA はまずこの MX レコードを DNS に問い合わせて配送先サーバーを決定します。
MTA
Mail Transfer Agent
メール転送エージェント
メールを実際に転送・配送するサーバーソフトウェア。Postfix / Sendmail / Microsoft Exchange などが代表例。送信側と受信側の両方に存在し、MX レコードを参照して次のホップへメールを届けます。
DSN
Delivery Status Notification
配送状態通知（RFC 3464）
配送状態を通知する仕組みの総称。成功・遅延・失敗の3種類があります。バウンスメール本文の標準フォーマットそのものを指し、Return-Path に設定されたアドレス宛に送られます。
NDR
Non-Delivery Report
配送失敗通知（DSN の一種）
DSN のうち「配送失敗」に絞った通知の俗称。バウンスメール・エラーメールとも呼ばれ、特に Microsoft Exchange 環境でよく使われる用語です。DSN が上位概念、NDR はその失敗ケースにあたります。
