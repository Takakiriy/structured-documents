IPv6 アドレス構造と割り当てガイド
128ビットアドレス空間の完全理解
1. IPv6アドレスの基本構造
IPv6アドレスは128ビットで構成され、上位64ビットのネットワークプレフィックスと下位64ビットのインターフェースIDに分かれています。
例: 2001:0db8:1234:5678:021a:2bff:fe3c:4d5e └─────┬─────┘└─┬─┘└────────┬────────┘ ISP割当 サブネット ホスト識別
2. よく使われるプレフィックス長
💡 重要ポイント
/64がIPv6の標準サブネットサイズです。すべてのLANセグメントに/64を割り当てることで、SLAAC（自動アドレス設定）が正常に動作し、管理もシンプルになります。
3. 階層的サブネット構成
サブネットIDの構造化例
2001:0db8:1234:SSFF::/64 ││││ ││└└─ フロア/VLAN番号（00-FF） └└─── サイト番号（00-FF） 具体例: :0101::/64 → サイト01、フロア01 :0102::/64 → サイト01、フロア02 :0201::/64 → サイト02、フロア01
4. インターフェースID（下位64ビット）の割り当て方法
① 手動割り当て（静的設定）
用途: サーバー、ルーター、ネットワーク機器
::1 # ゲートウェイ（ルーター） ::2, ::3 # 重要なサーバー ::10～::99 # サーバー群 ::dead:beef # 覚えやすいパターン
✓ 予測可能で管理しやすい
✓ DNSや証明書の設定に適している
② EUI-64（MACアドレスベース）
用途: 従来の自動生成方式
MACアドレス: 00:1a:2b:3c:4d:5e → インターフェースID: 021a:2bff:fe3c:4d5e 処理手順: 1. MACを分割: 00:1a:2b | 3c:4d:5e 2. 真ん中にff:fe挿入 3. 7ビット目を反転
⚠️ MACアドレスが露出（プライバシー問題）
⚠️ 現在はあまり推奨されない
③ プライバシー拡張（RFC 4941）
用途: クライアントPC（推奨）
例: a4b3:7f2c:8d91:e3f5 ↓ 定期的に変更（デフォルト24時間毎） 2c8f:91a3:b5d7:4e12
✓ ユーザーの追跡を困難にする
✓ Windows/macOS/Linuxでデフォルト有効
⚠️ ログ追跡が困難
④ 安定プライバシーアドレス（RFC 7217）
用途: IoT機器、企業内端末
hash(プレフィックス + ネットワークID + MACアドレス + シークレット) → 3f8a:c2d5:9e17:4b6a
✓ 同じネットワークでは同じアドレス
✓ ネットワークが変わると異なるアドレス
✓ 再接続時の安定性とプライバシーの両立
⑤ DHCPv6
用途: 企業環境での集中管理
連番方式: ::1000, ::1001, ::1002 ランダム方式: a3f4:b2c8:d5e1:7f90 MACベース予約: ::aaaa:bbbb:cccc
✓ 集中管理が可能
✓ 割り当てポリシーの柔軟な設定
⑥ SLAAC（自動設定）
用途: プラグアンドプレイ環境
ルーター広告でプレフィックスを受信 → 自動的にインターフェースIDを生成 → 設定不要で即座に通信可能
✓ /64必須
✓ 設定不要で簡単
✓ プライバシー拡張と組み合わせ可能
5. /64サブネット内のアドレス割り当て例
アドレス範囲 用途 備考
::0 サブネット自体 使用しない
::1 ゲートウェイ（ルーター） 標準的な慣例
::2 ～ ::ff インフラ機器 DNS、NTP、監視サーバー等
::100 ～ ::1ff サーバー 手動割り当て
::1000 ～ ::1fff DHCPv6プール 固定端末用
::8000:0:0:0 ～
::8fff:ffff:ffff:ffff		 SLAAC/プライバシー拡張 クライアント端末用
::ffff:ffff:ffff:ffff 予約 通常使用しない
🎯 ベストプラクティス
  • サーバー・インフラ: 手動で意味のある固定値を割り当て
  • 一般ユーザー: プライバシー拡張（自動変更）を使用
  • 企業環境: DHCPv6で集中管理
  • IoT機器: 安定プライバシーアドレスを使用
  • ドキュメント化: サブネット番号の意味を必ず文書化
  • 将来の拡張: 予約領域を確保しておく
6. まとめ
IPv6はアドレス空間が潤沢という前提で設計されています。そのため、アドレスをケチらず、標準化とシンプルさを優先した設計を心がけることが重要です。
IPv6のアドレス空間: /64サブネット = 約18京個（18,446,744,073,709,551,616個）のアドレス /48企業割当 = 65,536個の /64サブネット /32 ISP割当 = 約40億個の /48企業割当 地球上の全砂粒の数よりも多いアドレスが利用可能です！
