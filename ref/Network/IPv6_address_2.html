1. IPv6アドレスの表記法
IPv6アドレスは128ビットを16ビットずつ8つに":"（コロン）で区切った数値列を、16進数で表記します。
基本表記
ABCD:EF01:2345:6789:ABCD:EF01:2345:6789 2001:0DB8:0000:0000:0008:0800:200C:417A
省略ルール
省略表記 アドレスの種類
FF01::101 マルチキャストアドレス
2001:DB8::8:800:200C:417A ユニキャストアドレス
::1 ループバックアドレス
:: 未指定アドレス
プレフィックス表記
IPv4のCIDR表記と同様に、[IPv6アドレス]/[プレフィックス長] で表記します。
例: 2001:DB8::/32、2001:DB8:1234:5678::/64
2. IPv6アドレスの型
IPv6アドレスは上位ビットの値によって、複数の型に分類されます。
3. ユニキャストアドレスについて
3.1 グローバルユニキャストアドレスの範囲
2000::/3は「先頭3ビットが001」という意味です。これがグローバルユニキャストアドレスの範囲全体を表しており、2001:から始まるアドレスもこの範囲に含まれます。
2001:から始まる特定のアドレスブロック
プレフィックス 用途 説明
2001::/32 Teredo IPv4/IPv6トンネリング技術用
2001:db8::/32 文書・ドキュメント用 RFC 3849で定義。実際の通信には使用不可
2001:2::/48 ベンチマーク用 性能測定用途
その他の2001:... グローバルユニキャスト 通常のインターネットアドレス
階層的な包含関係
2000::/3 (グローバルユニキャスト全体) ⊃ 2000::/4 (2000:〜2FFF:の範囲) ⊃ 2001::/16 (2001:から始まるアドレス) ⊃ 2001::/32 (Teredo用) ⊃ 2001:db8::/32 (文書用) ⊃ 2001:2::/48 (ベンチマーク用)
例えば、2001:db8::1 というアドレスは：
  • ✅ 2000::/3 の範囲内（先頭3ビットが001）
  • ✅ 2001::/16 の範囲内
  • ✅ 2001:db8::/32 の範囲内（文書用）
  • ✅ グローバルユニキャストアドレス
なぜ2000::/3なのか？
IPv6のアドレス空間（128ビット）は非常に広大です。先頭3ビットを見ることで、アドレスの大まかな分類が可能です：
  • 000: 特殊用途（未指定、ループバック等）
  • 001: グローバルユニキャスト（2000::/3）← 最も重要
  • 010〜110: 将来の利用のため予約
  • 111: マルチキャスト（ff00::/8は実際には先頭8ビット）
3.2 インタフェース識別子（EUI-64）
IPv6のインタフェース識別子は、多くの場合MACアドレスから生成されます。48ビットのMACアドレスを64ビットに拡張する方法をModified EUI-64と呼びます。
7ビット目の反転（U/Lビット）
MACアドレスの先頭バイトの7ビット目はU/L（Universal/Local）ビットと呼ばれます。
・0 = ローカル管理アドレス
・1 = グローバルに一意（IEEE割り当て）
IPv6では意味を反転させるため、このビットを反転します。
3.3 ユニークローカルIPv6ユニキャストアドレス（ULA）
ULAは、組織内ネットワークで使用するプライベートアドレスです。IPv4のプライベートアドレス（10.0.0.0/8など）に相当しますが、より高度な仕組みを持っています。
ULAの特徴
  • Lビット: 0の場合は将来の利用のため予約（現在未使用）、1の場合はローカルで割り当て可能（fd00::/8）
  • グローバルID: 40ビットのランダム値を使用することで、異なる組織間での重複を最小化
  • インターネット経路広告禁止: インターネット上でのルーティングは行わない
  • ISP非依存: ISPを変更してもアドレス変更不要
4. マルチキャストアドレス
IPv6のマルチキャストアドレスは、複数のノードに対して同時にデータを送信するためのアドレスです。IPv6にはブロードキャストがなく、マルチキャストがその役割を果たします。
マルチキャストの特徴
  • スコープによる範囲制御: パケットの到達範囲を明確に定義できる
  • ブロードキャストの代替: IPv6にはブロードキャストがなく、マルチキャストで実現
  • 効率的な配信: 必要なノードだけが受信するため、ネットワーク負荷を軽減
  • 近隣探索に利用: IPv6の基本機能である近隣探索プロトコル（NDP）で使用
5. 特殊用途のIPv6アドレス
5.1 文書記述用アドレスプレフィックス
マニュアルや設定サンプルへの利用を想定した、文書記述専用のアドレスプレフィックスが定義されています。
文書用プレフィックス: 2001:db8::/32
例: 2001:db8:1234:5678::1 2001:db8:abcd:ef01::100 特徴: ・RFC 3849で定義 ・ドキュメント、マニュアル、設定例での使用専用 ・実際の通信には使用不可 ・本ガイドでもこのプレフィックスを使用
5.2 IPv4アドレス埋め込みIPv6アドレス
IPv4とIPv6の共存・移行技術で使用される特殊なアドレス形式です。
6. IPv6アドレス設計のポイント
💡 IPv6アドレス設計の重要ポイント
IPv6は単にアドレス空間を拡張しただけでなく、アドレスに構造を持たせ、自動設定を容易にし、将来の拡張性を確保した設計となっています。IPv4のアドレス不足問題を解決するだけでなく、ネットワーク管理の複雑さを軽減することも目的としています。
