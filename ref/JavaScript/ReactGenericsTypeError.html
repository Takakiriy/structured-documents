TypeScript ジェネリック型推論エラーの解決

エラーメッセージの例と読み方

Type '{ field: "email"; order: "asc"; }' is not assignable to type 'SortState<TEntity, "name" | "age">'. Types of property 'field' are incompatible. Type '"email"' is not assignable to type '"name" | "age"'.
エラーメッセージの読み方(1):
TypeScriptのエラーメッセージは階層構造になっており、上から順に原因を掘り下げていきます。

1行目: sort オブジェクト全体が SortState 型に合わない
→ 期待される型: SortState<TEntity, "name" | "age">（columnsから推論）

2行目: どのプロパティが問題か → field プロパティ

3行目: 下記

Type '"email"' is not assignable to type '"name" | "age"'
エラーメッセージの読み方(2):
Type '"email"' ← 実際に渡されている値の型（sort.fieldの値）

is not assignable to ← 「〜に代入できない」という意味

type '"name" | "age"' ← 期待されている型（columnsから推論されたTField型）

つまり、"email" という値は "name" | "age" というユニオン型には含まれていないため、型エラーになります。

どのプロパティから先に推論するかは実装の詳細であり、保証されていません。

問題のコードの型

以下の 3つの型定義すべてに TField という同じ型パラメーター（ジェネリクス関連）が使われています。
これにより、ContentTableColumn の field と SortState の field は同じ型でなければなりません。
#focus: TField, field
// Props の型定義 interface Props<TEntity, TField extends string> { columns: ContentTableColumn<TEntity, TField>[]; sort?: hooks.SortState<TEntity, TField>; } // ContentTableColumn の型定義 interface ContentTableColumn<TEntity, TField extends string> { field: TField; label: string; // ...その他のプロパティ } // SortState の型定義 namespace hooks { interface SortState<TEntity, TField extends string> { field: TField; order: "asc" | "desc"; } }

エラーケース ❌

✗ sort.field (SortState.field) が columns (ContentTableColumn[]) に含まれていない
const data = { columns: [ { field: "name" as const, label: "Name" }, { field: "age" as const, label: "Age" } ], sort: { field: "email" as const, order: "asc" } // ← エラー! } as const; <Component {...data} /> // エラー: Type '"email"' is not assignable to type '"name" | "age"'
何が起きているか:
TypeScript は columns から TField = "name" | "age" と推論します。
しかし sort.field は "email" であり、TField 型に含まれないためエラーになります。

解決法: sort.field を columns に含まれる値にする ✅

✓ sort.field が columns に含まれている
const data = { columns: [ { field: "name" as const, label: "Name" }, { field: "age" as const, label: "Age" } ], sort: { field: "name" as const, order: "asc" } // ✓ columns にある値 } as const; <Component {...data} /> // ✓ エラーなし!
正しい動作:
sort.field の値 "name" は columns に含まれているため、
TypeScript は TField = "name" | "age" として問題なく型推論できます。
まとめ
根本原因: columns と sort から推論される TField の型が矛盾している

解決方法: sort.field の値を columns に含まれるフィールド名にする

ポイント: TypeScript は両方のプロパティから同じ型パラメータを推論しようとするため、
どちらか一方だけに存在する値を使うとエラーになります。
